Kimi-first orchestration for real work.

oh-my-kimi turns Kimi Code CLI into a workflow-oriented environment with skills, setup, team coordination, and durable state — while keeping omx as a compatibility alias during the migration.

Get started View on GitHub Browse docs
A lightweight public site for the oh-my-kimi fork, built to mirror the project docs and link the CLI repo to a public home page.

What it gives you

Commands that feel like a product

Setup, doctor, help, team, plan, ralph, and related workflows are presented as a coherent system.

Durable project state

Plans, logs, memory, and mode tracking live under .omx/ so work survives across sessions.

Documentation that ships with the repo

Start here for getting started, compatibility boundaries, and the key docs pages.

Public-facing brand surface

The website is intentionally separate from the CLI repo so the public site can stay polished and easy to update.

Quick start

  1. npm install -g oh-my-kimi
  2. omk setup
  3. omk doctor
  4. omk or omx for compatibility

Compatibility boundary

The fork is Kimi-first, but some OMX behaviors are still compatibility-oriented. Read the compatibility page before assuming full parity.
