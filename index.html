A Kimi-first workflow layer for planning, execution, and verification.

oh-my-kimi turns Kimi Code CLI into a workflow-oriented environment for planning, execution, and verification. It adds skills, setup, coordinated team runs, and durable state, with a small compatibility layer for older workflows.

A lightweight public site for the oh-my-kimi fork, built to mirror the project docs and link the CLI repo to a public home page.

What it gives you

What this project does

It turns Kimi Code CLI into a workflow layer that helps you plan work, execute it in parallel, verify the result, and keep project state across sessions.

Durable project state

Plans, logs, memory, and mode tracking live in the project state directory so work survives across sessions.

Documentation that ships with the repo

Start here for getting started, compatibility boundaries, and the key docs pages.

Public-facing brand surface

The website is intentionally separate from the CLI repo so the public site can stay polished, easy to update, and easy to understand for new visitors.

Quick start

  1. npm install -g oh-my-kimi
  2. omk setup
  3. omk doctor
  4. omk is the primary command name; older workflows remain compatible during the migration.

Compatibility boundary

The fork is Kimi-first, but some older workflow behaviors are still compatibility-oriented. Read the compatibility page before assuming full parity.
