Migration boundary

What is fully Kimi-first and what is still compatibility-only?

oh-my-kimi is intentionally not a blind rename. Some surfaces are stable aliases while the port finishes.

Kimi-first

  • Primary package name: oh-my-kimi
  • Primary command name: omk
  • Public docs and website branding
  • Setup / doctor / help surfaces

Compatibility alias

  • omx remains available during migration
  • Historical mentions may still exist in archived docs
  • Some lower-level runtime terms may still say OMX internally

Unsupported until explicitly added

If a Kimi capability does not have a direct OMX equivalent, the safest path is to document it as a limitation instead of pretending parity exists.
