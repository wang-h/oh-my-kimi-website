Workflow shortcuts

Skills turn common tasks into repeatable flows.

These are the high-traffic skills that make the Kimi-first fork feel like a product rather than a pile of commands.

SkillWhat it doesWhen to use it
omk setupInstalls prompts, skills, and state scaffolding.First run or refresh.
omk doctorChecks install, config, and runtime health.Something looks broken.
ralplanCreates a consensus plan with planner / architect / critic review.Before a larger change.
ralphPersistent execution loop with verification.When you want completion, not just a suggestion.
teamRuns coordinated tmux workers for parallel execution.When the task splits into independent lanes.
hudSurfaces runtime state, turns, and active modes.When you want a live summary.

Planning

plan / ralplan help you scope work before code changes start.

Execution

ralph, team, and autopilot drive the work until it is verified.

Support

doctor, help, and hud make the runtime easier to understand.
