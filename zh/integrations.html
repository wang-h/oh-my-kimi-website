集成

把这个 shell 接到你的 Kimi CLI 工作环境里。

oh-my-kimi 最有价值的时候，往往不是孤立使用，而是和 MCP、通知、tmux team、GitHub 以及其他运行时能力一起工作。

MCP

让 Kimi CLI 会话可以围绕状态、记忆、代码智能和外部工具做结构化连接。

通知

把长流程的结果推到你已经在看的地方，而不是一直守着终端。

GitHub

把 CLI 仓库、网站仓库、release 文档和公开定位统一起来。

tmux runtime

任务大到一条终端线不够时，就用 team 和 HUD 把运行时扩展开。
