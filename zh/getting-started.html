开始这里

四步把 Kimi CLI 变成一个更完整的工作流外壳。

如果你的理解是“我想给 Kimi CLI 装一个类似 oh-my-codex 的东西”，那就按这条路径走： 安装、setup、doctor、启动 shell。

1. 安装

npm install -g oh-my-kimi

安装这个面向 Kimi CLI 的增强层。

2. 初始化

omk setup

写入提示词、技能、配置，以及本地 .omk/ 状态目录。

3. 检查环境

omk doctor

确认增强层、Kimi CLI 路径和运行时环境都正常。

4. 启动 shell

omk

从这里开始，你是在通过 oh-my-kimi 使用 Kimi CLI。

建议先跑的流程

$deep-interview "澄清任务"
$ralplan "批准实施计划"
$ralph "持续推进到完成"

这是最能感受到 oh-my-kimi 和“单纯多一个命令”区别的地方。
