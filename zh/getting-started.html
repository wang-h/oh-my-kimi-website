开始这里

几分钟内从安装到第一个工作流。

oh-my-kimi 让 CLI 保持 Kimi-first，同时在迁移期间保留 旧别名 兼容过渡。

1. 安装

npm install -g oh-my-kimi

如果你已经装了旧别名，它仍然可用。

2. 初始化

omk setup

这会写入 Kimi-first 默认值、提示词、技能和项目状态。

3. 检查环境

omk doctor

当安装或运行看起来不对时，用 doctor 检查。

4. 开始工作

omk

迁移期间 旧别名仍然可用。
