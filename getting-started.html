Start here

Get from install to first workflow in a few minutes.

oh-my-kimi keeps the CLI path Kimi-first while preserving omx as an alias during migration.

1. Install

npm install -g oh-my-kimi

If you already have the alias installed, omx still works.

2. Initialize

omk setup

This sets up the Kimi-first defaults, prompts, skills, and project state.

3. Check the environment

omk doctor

Use doctor when install or runtime behavior seems off.

4. Start working

omk

For compatibility, omx remains available while the migration finishes.

Recommended first check

After setup, read compatibility notes to understand which behaviors are Kimi-first and which are still compatibility aliases.
