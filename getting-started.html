Start here

Install the Kimi CLI enhancement layer in four steps.

If your mental model is “I want something like oh-my-codex, but for Kimi CLI,” this is the shortest path. Install the package, run setup, verify the environment, then launch into the workflow shell.

1. Install

npm install -g oh-my-kimi

Install the package that wraps Kimi CLI with the oh-my-kimi workflow layer.

2. Initialize

omk setup

Scaffold prompts, skills, config, and local .omk/ runtime state.

3. Check the environment

omk doctor

Confirm the enhancement layer, Kimi CLI path, and supporting runtime are healthy.

4. Launch the shell

omk

From there, you are using Kimi CLI through the oh-my-kimi shell.

Recommended first workflow

/skill:deep-interview clarify the task
/skill:ralplan approve the implementation plan
/skill:ralph drive it to completion

That is the fastest way to feel what makes oh-my-kimi more than a plain command wrapper. Use /skill: entrypoints by default inside Kimi CLI.
